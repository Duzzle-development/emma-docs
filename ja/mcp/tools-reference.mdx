## タスク実行

### emma_run_task

emmaでタスクを実行します。

<ParamField path="prompt" type="string" required>
  実行するタスクの説明。具体的かつ明確に記述してください。
  
  **例:**
  - `「今日の天気をSlackの#generalに投稿して」`
  - `「最新のGitHub issueをまとめて」`
  - `「明日のミーティングについてチームにリマインダーメールを送って」`
</ParamField>

<ParamField path="mode" type="string" default="normal">
  タスクの実行モード。
  
  | 値 | 説明 |
  |----|------|
  | `normal` | 標準実行（デフォルト） |
  | `deep_thinking` | 複雑なタスク向けの拡張推論 |
  | `ask` | 情報収集モード |
</ParamField>

<ParamField path="timezone" type="string" default="Asia/Tokyo">
  IANA形式のタイムゾーン（例: `America/New_York`, `Europe/London`）。
</ParamField>

<Tabs>
  <Tab title="リクエスト例">
    ```json
    {
      "prompt": "今日の天気予報をSlackの#generalに投稿して",
      "mode": "normal",
      "timezone": "Asia/Tokyo"
    }
    ```
  </Tab>
  <Tab title="レスポンス例">
    ```json
    {
      "success": true,
      "task_id": "task_abc123",
      "message": "タスクが正常に実行されました"
    }
    ```
  </Tab>
</Tabs>

### emma_session_interact

既存のemmaセッションと対話します。この統合ツールを使用して、emmaセッションの監視、通信、制御を行うことができます。

<Info>
**最適な用途:** タスク進捗の監視、実行中のタスクへのフォローアップ指示の送信、不要になったタスクのキャンセル、Agent-to-Agent（A2A）通信パターン。

**推奨されない用途:** 新しいタスクの開始（代わりに`emma_run_task`を使用）、APIヘルスチェック（代わりに`emma_health_check`を使用）。

**最適なワークフロー:** `emma_run_task`でタスクを開始し、`conversation_id`をメモしてから、このツールを使用して進捗を監視したり、追加の指示を送信したりします。
</Info>

<ParamField path="session_id" type="string" required>
  `emma_run_task`から返されるセッションID（`conversation_id`）。対話するセッションを識別します。
</ParamField>

<ParamField path="action" type="string" required>
  実行するアクション。
  
  | 値 | 説明 |
  |----|------|
  | `get_status` | 現在のステップ、進捗率、アクティブなエージェント、最後のメッセージを含む詳細なセッションステータスを取得 |
  | `send_message` | 追加の指示やコンテキストを提供するフォローアップメッセージを送信 |
  | `cancel` | 実行中のセッションをキャンセル |
</ParamField>

<ParamField path="message" type="string">
  メッセージ内容（`send_message`アクションの場合は必須）。実行中のタスクに追加の指示やコンテキストを提供します。
</ParamField>

**レスポンスフィールド:**

| フィールド | 説明 |
|-----------|------|
| `status` | セッション状態（`pending`, `running`, `completed`, `cancelled`, `failed`） |
| `status_message` | 現在の状態の人間が読める説明 |
| `task` | タスク進捗（`task_id`, `current_step`, `total_steps`, `completed_steps`, `plan_summary`） |
| `current_agent` | 現在実行中のエージェント情報（`name`, `action`, `step_title`） |
| `progress` | 進捗率と経過時間 |
| `can_send_message` | セッションがメッセージを受け付けるかどうか |
| `action_result` | `send_message`または`cancel`アクションの結果 |

<Tabs>
  <Tab title="ステータス取得">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "get_status"
    }
    ```
    
    **レスポンス:**
    ```json
    {
      "status": "running",
      "status_message": "タスクは進行中です",
      "task": {
        "task_id": "task_xyz789",
        "current_step": 2,
        "total_steps": 5,
        "completed_steps": 1,
        "plan_summary": "データ収集、結果分析、レポート生成"
      },
      "current_agent": {
        "name": "DataAnalyzer",
        "action": "analyzing",
        "step_title": "Q3売上データを分析中"
      },
      "progress": {
        "percentage": 40,
        "elapsed_time": "00:02:15"
      },
      "can_send_message": true
    }
    ```
  </Tab>
  <Tab title="メッセージ送信">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "send_message",
      "message": "レポートにQ4のデータも含めてください"
    }
    ```
    
    **レスポンス:**
    ```json
    {
      "status": "running",
      "action_result": {
        "success": true,
        "message": "メッセージがセッションに配信されました"
      }
    }
    ```
  </Tab>
  <Tab title="キャンセル">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "cancel"
    }
    ```
    
    **レスポンス:**
    ```json
    {
      "status": "cancelled",
      "action_result": {
        "success": true,
        "message": "セッションがキャンセルされました"
      }
    }
    ```
  </Tab>
</Tabs>

## オートメーション管理

### emma_create_automation

新しいスケジュールオートメーションを作成します。

<ParamField path="name" type="string" required>
  オートメーションの名前。
  
  **例:** `「デイリーヘルスダイジェスト」`, `「週次レポート」`, `「1時間ごとのステータスチェック」`
</ParamField>

<ParamField path="prompt" type="string" required>
  emmaがスケジュールに従って実行するタスクのプロンプト。
</ParamField>

<ParamField path="cron_expression" type="string" required>
  スケジュールを定義するcron式。
  
  **一般的なパターン:**
  | 式 | 説明 |
  |----|------|
  | `0 9 * * *` | 毎日9:00 |
  | `0 9 * * 1` | 毎週月曜9:00 |
  | `0 */2 * * *` | 2時間ごと |
  | `0 9 1 * *` | 毎月1日9:00 |
</ParamField>

<ParamField path="timezone" type="string" default="Asia/Tokyo">
  IANA形式のスケジュール用タイムゾーン。
</ParamField>

<ParamField path="enabled" type="boolean" default={true}>
  オートメーションをすぐに有効にするかどうか。
</ParamField>

<Tabs>
  <Tab title="リクエスト例">
    ```json
    {
      "name": "デイリースタンドアップリマインダー",
      "prompt": "Slackの#teamにスタンドアップリマインダーを送信",
      "cron_expression": "0 9 * * 1-5",
      "timezone": "Asia/Tokyo",
      "enabled": true
    }
    ```
  </Tab>
  <Tab title="レスポンス例">
    ```json
    {
      "success": true,
      "automation_id": "sched_abc123",
      "name": "デイリースタンドアップリマインダー",
      "next_run": "2024-01-16T09:00:00+09:00"
    }
    ```
  </Tab>
</Tabs>

## Cron式リファレンス

Cron式は標準の5フィールド形式に従います：

```
┌───────────── 分 (0-59)
│ ┌───────────── 時 (0-23)
│ │ ┌───────────── 日 (1-31)
│ │ │ ┌───────────── 月 (1-12)
│ │ │ │ ┌───────────── 曜日 (0-6, 日曜=0)
│ │ │ │ │
* * * * *
```

### 一般的な例

| 式 | 説明 |
|----|------|
| `0 9 * * *` | 毎日9:00 |
| `0 9 * * 1-5` | 平日9:00 |
| `0 9 * * 0` | 毎週日曜9:00 |
| `0 */2 * * *` | 2時間ごと |
| `30 8 * * *` | 毎日8:30 |
| `0 9 1 * *` | 毎月1日9:00 |
| `0 9 * * 1` | 毎週月曜9:00 |
| `0 0 * * *` | 毎日0:00 |

### 特殊文字

| 文字 | 説明 | 例 |
|------|------|-----|
| `*` | 任意の値 | `* * * * *`（毎分） |
| `,` | リスト区切り | `0 9,12,18 * * *`（9時、12時、18時） |
| `-` | 範囲 | `0 9 * * 1-5`（月曜から金曜） |
| `/` | ステップ | `*/15 * * * *`（15分ごと） |
