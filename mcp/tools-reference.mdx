## Task Execution

### emma_run_task

Execute a task on emma.

<ParamField path="prompt" type="string" required>
  The task description to execute. Be specific and clear.
  
  **Examples:**
  - `"Post today's weather to Slack #general"`
  - `"Summarize the latest GitHub issues"`
  - `"Send a reminder email to the team about tomorrow's meeting"`
</ParamField>

<ParamField path="mode" type="string" default="normal">
  Execution mode for the task.
  
  | Value | Description |
  |-------|-------------|
  | `normal` | Standard execution (default) |
  | `deep_thinking` | Extended reasoning for complex tasks |
  | `ask` | Information gathering mode |
</ParamField>

<ParamField path="timezone" type="string" default="Asia/Tokyo">
  Timezone in IANA format (e.g., `America/New_York`, `Europe/London`).
</ParamField>

<Tabs>
  <Tab title="Example Request">
    ```json
    {
      "prompt": "Post today's weather forecast to Slack #general",
      "mode": "normal",
      "timezone": "Asia/Tokyo"
    }
    ```
  </Tab>
  <Tab title="Example Response">
    ```json
    {
      "success": true,
      "task_id": "task_abc123",
      "message": "Task executed successfully"
    }
    ```
  </Tab>
</Tabs>

### emma_session_interact

Interact with an existing emma session. This unified tool allows you to monitor, communicate with, and control emma sessions.

<Info>
**Best for:** Monitoring task progress, sending follow-up instructions to running tasks, cancelling tasks that are no longer needed, Agent-to-Agent (A2A) communication patterns.

**Not recommended for:** Starting new tasks (use `emma_run_task` instead), checking API health (use `emma_health_check`).

**Optimal workflow:** Start a task with `emma_run_task`, note the `conversation_id`, then use this tool to monitor progress or send additional instructions.
</Info>

<ParamField path="session_id" type="string" required>
  The session ID (`conversation_id`) returned from `emma_run_task`. This identifies the session to interact with.
</ParamField>

<ParamField path="action" type="string" required>
  The action to perform.
  
  | Value | Description |
  |-------|-------------|
  | `get_status` | Get detailed session status including current step, progress percentage, active agent, and last message |
  | `send_message` | Send a follow-up message to provide additional instructions or context |
  | `cancel` | Cancel the running session |
</ParamField>

<ParamField path="message" type="string">
  Message content (required for `send_message` action). Provide additional instructions or context for the running task.
</ParamField>

**Response Fields:**

| Field | Description |
|-------|-------------|
| `status` | Session state (`pending`, `running`, `completed`, `cancelled`, `failed`) |
| `status_message` | Human-readable description of current state |
| `task` | Task progress (`task_id`, `current_step`, `total_steps`, `completed_steps`, `plan_summary`) |
| `current_agent` | Currently executing agent info (`name`, `action`, `step_title`) |
| `progress` | Progress percentage and elapsed time |
| `can_send_message` | Whether the session accepts messages |
| `action_result` | Result of `send_message` or `cancel` action |

<Tabs>
  <Tab title="Get Status">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "get_status"
    }
    ```
    
    **Response:**
    ```json
    {
      "status": "running",
      "status_message": "Task is in progress",
      "task": {
        "task_id": "task_xyz789",
        "current_step": 2,
        "total_steps": 5,
        "completed_steps": 1,
        "plan_summary": "Gathering data, analyzing results, generating report"
      },
      "current_agent": {
        "name": "DataAnalyzer",
        "action": "analyzing",
        "step_title": "Analyzing Q3 sales data"
      },
      "progress": {
        "percentage": 40,
        "elapsed_time": "00:02:15"
      },
      "can_send_message": true
    }
    ```
  </Tab>
  <Tab title="Send Message">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "send_message",
      "message": "Please also include the Q4 data in the report"
    }
    ```
    
    **Response:**
    ```json
    {
      "status": "running",
      "action_result": {
        "success": true,
        "message": "Message delivered to session"
      }
    }
    ```
  </Tab>
  <Tab title="Cancel">
    ```json
    {
      "session_id": "conv_abc123",
      "action": "cancel"
    }
    ```
    
    **Response:**
    ```json
    {
      "status": "cancelled",
      "action_result": {
        "success": true,
        "message": "Session has been cancelled"
      }
    }
    ```
  </Tab>
</Tabs>

## Automation Management

### emma_create_automation

Create a new scheduled automation.

<ParamField path="name" type="string" required>
  Name for the automation.
  
  **Examples:** `"Daily Health Digest"`, `"Weekly Report"`, `"Hourly Status Check"`
</ParamField>

<ParamField path="prompt" type="string" required>
  The task prompt that emma will execute on schedule.
</ParamField>

<ParamField path="cron_expression" type="string" required>
  Cron expression defining the schedule.
  
  **Common patterns:**
  | Expression | Description |
  |------------|-------------|
  | `0 9 * * *` | Every day at 9:00 AM |
  | `0 9 * * 1` | Every Monday at 9:00 AM |
  | `0 */2 * * *` | Every 2 hours |
  | `0 9 1 * *` | First day of every month at 9:00 AM |
</ParamField>

<ParamField path="timezone" type="string" default="Asia/Tokyo">
  Timezone for the schedule in IANA format.
</ParamField>

<ParamField path="enabled" type="boolean" default={true}>
  Whether the automation should be enabled immediately.
</ParamField>

<Tabs>
  <Tab title="Example Request">
    ```json
    {
      "name": "Daily Standup Reminder",
      "prompt": "Send a standup reminder to Slack #team",
      "cron_expression": "0 9 * * 1-5",
      "timezone": "Asia/Tokyo",
      "enabled": true
    }
    ```
  </Tab>
  <Tab title="Example Response">
    ```json
    {
      "success": true,
      "automation_id": "sched_abc123",
      "name": "Daily Standup Reminder",
      "next_run": "2024-01-16T09:00:00+09:00"
    }
    ```
  </Tab>
</Tabs>

## Cron Expression Reference

Cron expressions follow the standard 5-field format:

```
┌───────────── minute (0-59)
│ ┌───────────── hour (0-23)
│ │ ┌───────────── day of month (1-31)
│ │ │ ┌───────────── month (1-12)
│ │ │ │ ┌───────────── day of week (0-6, Sunday=0)
│ │ │ │ │
* * * * *
```

### Common Examples

| Expression | Description |
|------------|-------------|
| `0 9 * * *` | Every day at 9:00 AM |
| `0 9 * * 1-5` | Weekdays at 9:00 AM |
| `0 9 * * 0` | Every Sunday at 9:00 AM |
| `0 */2 * * *` | Every 2 hours |
| `30 8 * * *` | Every day at 8:30 AM |
| `0 9 1 * *` | First day of month at 9:00 AM |
| `0 9 * * 1` | Every Monday at 9:00 AM |
| `0 0 * * *` | Every day at midnight |

### Special Characters

| Character | Description | Example |
|-----------|-------------|---------|
| `*` | Any value | `* * * * *` (every minute) |
| `,` | List separator | `0 9,12,18 * * *` (9 AM, 12 PM, 6 PM) |
| `-` | Range | `0 9 * * 1-5` (Monday to Friday) |
| `/` | Step | `*/15 * * * *` (every 15 minutes) |
